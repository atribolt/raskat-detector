Для админ панели предусмотрена работа с пользователями. Этот механизм позволяет прокинуть админ-панель на публичный домен и предоставить друзьям возможность просматривать работу вашего детектора.

## Роли пользователей

Пока предусмотрены только две роли:

-  `admin` - управление детектором.

-  `viewer` - просмотр сигналов. Нельзя выполнять административные функции

## Начальные пользователи

По умолчанию лаунчер при первичном запуске на детекторе создает двух пользователей:

-  `admin` - администратор (по умолчанию пароль `admin`)

-  `user` - пользователь для просмотра сигналов (по умолчанию пароль `user`)

## Управление пользователями

:::note 

Для управления пользователями сейчас необходимо подключаться к детектору по `ssh`

:::

Общая команда работы с пользователями:

```sh
launcher -U <login> -P <pass> -R <role>
```

В этой команде:

-  `<login>` - логин пользователя. Если пользователь существует, происходит обновление пользователя. Если пользователь с таким логином не существует, пользователь будет создан.

   :::lab 

   При создании пользователя требуется передать все параметры `-U` `-P` и `-R`

   :::

-  `<pass>` - новый пароль пользователя

-  `<role>` - новая роль пользователя. Может быть:

   -  `admin` - управление детектором

   -  `viewer` - только просмотр

:::note 

**После изменения/добавления/удаления пользователей требуется его перезапуск**

`systemctl restart launcher`

:::

### Создание пользователя

```bash
launcher -U "test" -P "veryhardpassword" -R "viewer"
```

### Сменить пароль пользователю

```bash
launcher -U "test" -P "newveryhardpassword"
```

### Сменить роль пользователю

```bash
launcher -U "test" -R "admin"
```

### Удалить пользователя

```sh
launcher -U "test" --delete
```
