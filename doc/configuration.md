# Конфигурация ПО

## Конфигурация детектора

- Конфигурационный файл `/etc/detector/detector.conf`. Файл имеет простой формат: `key = value`
  
  Конфиг файл для утилиты обнаружения пиков
  
  Все после `#` в любом месте, считается комментарием и не обрабатывается при разборе
  
  Типы данных:
    - строка `[str]` - `"d'a'ta"` или `'d"a"ta'`
    - число `[num]` - `0.0`
    - булево `[bool]` - `true` или `false`
  

### Параметры

|Параметр|Тип данных|Значение по умолчанию|Описание|
|---|:---:|:---:|---|
|`EmaEnable`|`bool`|`true`|Включение динамического порога|
|`Threshold`|`num`|`0.2`|Cтатический порог в вольтах. Используется только при `EmaEnable = false`|
|`EmaReaction`|`num`|`0.02`|Настройка динамического порога `EMA`: `EMA = EmaReaction * Xn + (1 - (2 / (EmaWindow + 1) * EMA(n-1))`. Кэфициент нарастания порога. Чем больше, тем быстрее нарастает порог, тем больше пиков будем пропускать. Используется только при `EmaEnable = true`|
|`EmaWindow`| `num`|`125`|Длинна окна анализа порога. Чем длиннее окно, тем медленнее реакция на изменения (больше пиков пропустим)|
|`PPSRequired`|`bool`|`true`|Требуется ли синхронизация по PPS. Захват сигналов не будет начат, пока модуль GPS не синхронизируется со спутниками|
|`DefaultLongitude`|`num`|`0.0`|Значение долготы по умолчанию. Используется, когда синхронизация по GPS не выполнена, или выполнена не полностью|
|`DefaultLatitude`|`num`|`0.0`|Значение широты по умолчанию. Используется, когда синхронизация по GPS не выполнена, или выполнена не полностью|
|`DefaultAltitude`|`num`|`0.0`|Значение высоты над уровнем моря (`HAE`) по умолчанию. Используется, когда синхронизация по GPS не выполнена, или выполнена не полностью|


## Конфигурация лаунчера

`raskat-launcher` не имеет файлов конфигурации

### Управление пользователями

По умолчанию лаунчер создает двух пользователей:
- `admin` - администратор (по умолчанию пароль `admin`)
- `user` - просмотр сигналов (по умолчанию пароль `user`)

> Для управления пользователями сейчас необходимо подключаться на `OrangePi One`

Общая команда работы с пользователями:
```sh
launcher -U <login> -P <pass> -R <role>
```

В этой команде:
- `<login>` - логин пользователя. Если пользователь существует, происходит обновление пользователя. Если пользователь с таким логином не существует, пользователь будет создан.
  > При создании пользователя требуется передать все параметры -U -P и -R
- `<pass>` - новый пароль пользователя
- `<role>` - новая роль пользователя. Может быть:
  - `admin` - управление детектором
  - `viewer` - только просмотр

> **После изменения/добавления/удаления пользователей требуется его перезапуск `systemctl restart launcher`**

#### Примеры команд

```sh
# Затягиваем файл с паролями, чтобы в истории не сохранялись пароли
source /tmp/credentials

# Создание нового пользователя
launcher -U test -P "${TEST_PASS}" -R viewer

# Сменить пароль пользователю
launcher -U user -P "${USER_PASS}"

# Сменить роль пользователю
launcher -U test -R admin

# Удалить пользователя
launcher -U test --delete

# После изменения пользователей, требуется перезапуск лаунчера
systemctl restart launcher
```
